# spec_market_bot_20260330

## Summary

为仓库补充一个每天 `09:25`、`13:25`、`18:25` 触发的定时任务，抓取 `XAU/USD`、`XAG/USD`、`WTI`、`ETH/USD`、`NDX`、`SPX`、`USDX` 与 `SH` 实时报价，并将格式化后的行情消息推送到 QQ 机器人。

在后续迭代中，消息体扩展为“价格 + AI 新闻 Top 10 + 财经新闻 Top 10”的组合播报，并要求输出中文摘要，同时持续优化新闻源质量、去重与防截断表现。

## Goals

- 支持稳定拉取 `XAU/USD`、`XAG/USD`、`WTI`、`ETH/USD`、`NDX`、`SPX`、`USDX` 与 `SH` 报价。
- 支持至少一种可落地的 QQ 机器人推送方式。
- 提供最小化的本地定时执行方案与可复制的配置模板。
- 提供可复制的 Linux 服务器部署方案，并确保在 OneBot 暂不可达时保留清晰预检日志。
- 支持在 Linux 服务器上把 OneBot 与行情机器人一起部署，并将 OneBot 管理面板限制在本机回环地址。
- 支持抓取 AI 新闻前 `10` 条、财经新闻前 `10` 条，并以中文摘要形式输出。
- 支持在 Twelve Data 当前套餐不覆盖 `NDX`、`SPX`，且服务器出口对新浪全球指数接口返回 `Forbidden` 时，自动改走服务器可用的替代源。
- 支持把同一轮播报同时发往群与多个私聊 QQ 接收者。
- 支持按上海时区记录“当天已发送新闻”，避免 `09:25`、`13:25`、`18:25` 三次推送之间重复发送同一条 AI 或财经新闻。
- 支持在财经新闻质量不理想时，优先改走标题型数据源并保留东方财富回退链路。
- 在消息过长时自动拆分为多条连续推送，避免 QQ 官方机器人发送失败。

## Non-goals

- 不在本次内接入复杂的消息卡片、图表或数据库落盘。
- 不实现小程序页面内展示或后台管理界面。

## User Flows

- 操作: 配置环境变量和机器人目标
  反馈: 脚本可在本地直接 dry-run 或真实推送
  回退: 任一配置缺失时脚本明确报错
- 操作: 在每天 `09:25`、`13:25`、`18:25` 执行脚本
  反馈: QQ 侧收到包含八个价格品种与两组新闻的完整播报消息
  回退: 推送失败时 cron 日志保留错误信息
- 操作: 将脚本部署到 Linux 服务器并交给 cron 托管
  反馈: 服务器可独立执行 dry-run，且定时任务在 OneBot 不可达时先写入明确预检日志
  回退: 修正 `ONEBOT_HTTP_URL` 后无需重新部署代码，只需更新 `.env`
- 操作: 在服务器自建 NapCat 作为 OneBot 服务
  反馈: 服务器本机出现 `127.0.0.1:3000` 与 `127.0.0.1:6099` 监听，且可通过二维码完成 QQ 登录
  回退: 如果暂时不登录 QQ，行情 cron 会继续保留预检日志，不会把整套部署打坏

## Edge Cases

- 行情源返回错误或空价格
- AI 中文 feed 或标题型财经新闻源临时失败或结果不足 10 条
- 机器人鉴权失败或消息目标 ID 配置错误
- QQ 官方机器人场景下因平台主动消息限制导致发送失败
- OneBot 当前未部署在同一台服务器，导致 `http://127.0.0.1:3000` 不可达
- NapCat 首次部署后需要人工扫码授权 QQ 登录，未登录前 OneBot HTTP 接口可能无法真正返回业务状态
- `第一财经`、`36氪` 或 `CNBC` 页面结构调整，导致标题或行情解析规则需要更新
- `CNBC` 页面临时不可用时，需要退回到只保证价格可用的兜底源
- 同一事件被多个中文科技源重复报道，需要做近似去重而不是只看完全相同标题
- 当天后续时段如果候选新闻都已在更早一轮推送中过，需要明确提示“今天暂无新的新闻”，而不是重复旧内容

## Acceptance Criteria

- dry-run 能输出完整格式化消息
- 可通过环境变量切换 OneBot 与 QQ 官方机器人模式
- README 含 cron 模板、配置步骤与限制说明
- dry-run 输出包含八个价格品种、AI Top 10、财经 Top 10
- 新闻区块超出单条消息长度时自动拆分，并保持标题/编号可读
- `NDX`、`SPX` 在 Twelve Data 不可用且服务器访问新浪被拦截时，仍能通过 `CNBC` 或兜底源返回可用价格
- 财经新闻优先使用标题型源，避免大面积残句、重复和公告腔
- `ONEBOT_EXTRA_TARGETS` 支持同时配置多个群和私聊接收者，且不会与主目标重复发送
- 同一天第二次、第三次推送不会重复发送更早一轮已经发过的新闻；如果没有新增新闻，会明确显示“今天暂无新的新闻”
- 服务器侧提供独立 Node 运行时、包装脚本和 cron 配置，且 OneBot 不可达时能在日志中输出明确提示
- 服务器侧支持把 NapCat 绑定到 `127.0.0.1:3000/3001/6099`，并生成二维码供人工完成 QQ 登录
